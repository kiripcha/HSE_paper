# Подробная документация модулей и методов

Документ содержит описание всех публичных функций, классов и методов пакета
`crypto_risk` с примерами использования. Сгруппировано по модулям. Для каждой
сущности приводятся сигнатура, параметры, возвращаемое значение и пример.

## 1. config.py

Хранит параметры по умолчанию. Импорт:

```python
from crypto_risk.config import RiskConfig, RANDOM_SEED, DEFAULT_UNIVERSE
```

### Класс `RiskConfig`

Конфигурация, передаваемая в `RiskEngine`. Все поля имеют значения по
умолчанию.

| Поле | Тип | По умолчанию | Назначение |
|---|---|---|---|
| `universe` | tuple\[str, ...\] | вселенная по умолчанию | Список тикеров |
| `quote` | str | "USDT" | Базовая котировка |
| `market_proxy` | str | "BTC" | Тикер «рыночного индекса» |
| `var_confidence` | float | 0.99 | Уровень доверия VaR |
| `es_confidence` | float | 0.975 | Уровень доверия ожидаемых потерь |
| `horizons` | tuple\[int, ...\] | (1, 10) | Горизонты оценки в днях |
| `ewma_lambda` | float | 0.94 | Параметр сглаживания EWMA |
| `rolling_window` | int | 252 | Окно скользящих оценок |
| `trading_days_year` | int | 365 | Множитель аннуализации |
| `seed` | int | 42 | Случайное семя |
| `maker_fee` | float | 0.0002 | Комиссия пассивной заявки |
| `taker_fee` | float | 0.0005 | Комиссия активной заявки |
| `spread_mult` | float | 3.0 | Множитель спреда в формуле LVaR |
| `capital` | float | 1 000 000 | Размер капитала |

Пример:

```python
from crypto_risk import RiskConfig

cfg = RiskConfig(
    universe=("BTC", "ETH", "SOL"),
    var_confidence=0.99,
    horizons=(1, 5, 10),
    capital=5_000_000,
)
```

### Константы и функции модуля

`RANDOM_SEED: int = 42` — глобальное случайное семя.
`TRADING_DAYS_YEAR: int = 365` — множитель аннуализации.
`DEFAULT_UNIVERSE: tuple[str, ...]` — десять тикеров вселенной по умолчанию.
`QUOTE_ASSET: str = "USDT"`, `MARKET_PROXY: str = "BTC"`.

```python
from crypto_risk.config import to_symbol
to_symbol("BTC")            # "BTC/USDT"
to_symbol("ETH", "USD")     # "ETH/USD"
```

## 2. data/sources.py

Простой загрузчик с одной биржи и синтетический генератор.

### `fetch_ohlcv(base, start, end, timeframe="1d", quote="USDT", exchange_id="binance")`

Загружает свечи одного инструмента.

| Параметр | Тип | Назначение |
|---|---|---|
| `base` | str | Тикер актива, например "BTC" |
| `start`, `end` | str | Даты в формате "YYYY-MM-DD" |
| `timeframe` | str | "1m", "5m", "1h", "1d" |
| `quote` | str | Базовая котировка |
| `exchange_id` | str | Идентификатор биржи в `ccxt` |

Возвращает `pandas.DataFrame` со столбцами `open`, `high`, `low`, `close`,
`volume` и временным индексом в UTC.

```python
from crypto_risk.data.sources import fetch_ohlcv
df = fetch_ohlcv("BTC", "2024-01-01", "2024-02-01", "1h")
print(df.head())
```

### `fetch_order_book_snapshot(base, depth=50, quote="USDT", exchange_id="binance")`

Возвращает мгновенный снимок стакана.

| Возвращаемый ключ | Тип | Назначение |
|---|---|---|
| `bids`, `asks` | numpy.ndarray | Массивы пар цена-объём |
| `best_bid`, `best_ask` | float | Лучшие котировки |
| `mid` | float | Средняя цена |
| `spread`, `rel_spread` | float | Абсолютный и относительный спред |
| `timestamp` | pandas.Timestamp | Момент снимка |

```python
from crypto_risk.data.sources import fetch_order_book_snapshot
book = fetch_order_book_snapshot("BTC", depth=20)
print(f"Средняя цена {book['mid']}, относительный спред "
      f"{book['rel_spread']*1e4:.2f} б.п.")
```

### `synthetic_price_panel(universe, start, end, freq="D", seed=42)`

Детерминированный генератор синтетических панелей цен. Используется при
отсутствии сети.

```python
from crypto_risk.data.sources import synthetic_price_panel
prices = synthetic_price_panel(
    ("BTC", "ETH"), "2021-01-01", "2024-01-01", seed=42)
```

### Класс `CryptoDataLoader`

Загрузчик с кэшем CSV. Конструктор:
`CryptoDataLoader(universe, quote="USDT", exchange_id="binance", cache_dir=..., mode="auto")`.

| Поле/атрибут | Назначение |
|---|---|
| `mode` | "auto" (с сетью, фолбэк на синтетику), "exchange" (без фолбэка), "synthetic" (только синтетика) |
| `data_origin` | После загрузки — "exchange" или "synthetic" |
| `source_used` | Текстовое описание использованного источника |

Методы:
* `load_close_panel(start, end, timeframe="1d", use_cache=True)` — таблица цен
  закрытия. Колонки — тикеры.
* `load_ohlcv(base, start, end, timeframe="1d")` — полные OHLCV одного актива.

```python
from crypto_risk.data import CryptoDataLoader
ld = CryptoDataLoader(("BTC", "ETH"), mode="auto")
panel = ld.load_close_panel("2021-01-01", "2025-01-01", "1d")
print(ld.data_origin)
```

## 3. data/parsers.py

Мультиисточниковый сборщик с фолбэком и дозаполнением пропусков.

### Источники

Каждый источник — класс с атрибутом `name` и методом
`fetch_close(base, start, end) -> pandas.Series` (ряд цен закрытия в долларах
с временным индексом UTC).

| Класс | Глубина истории | Ключ |
|---|---|---|
| `CryptoCompareSource` | с 2010 года | не нужен |
| `YahooSource` | с 2014–2015 | не нужен |
| `CoinGeckoSource` | по ограничениям тарифа | не нужен |
| `BinanceCcxtSource` | с 2017 года | не нужен |

### Класс `MultiSourceCryptoLoader`

Главный загрузчик. Конструктор:
`MultiSourceCryptoLoader(universe, sources=None, mode="auto", cache_dir=...)`.

| Метод | Возвращает |
|---|---|
| `load_close_panel(start, end, use_cache=True)` | Таблица цен закрытия |
| `coverage_report()` | Таблица «актив → основной источник, кто дозаполнял» |

После загрузки доступны атрибуты `data_origin` и `source_used`.

```python
from crypto_risk.data import MultiSourceCryptoLoader, TOP10_POPULAR

ld = MultiSourceCryptoLoader(TOP10_POPULAR, mode="auto")
prices = ld.load_close_panel("2015-01-01", "2025-01-01")

print(ld.coverage_report())
print(f"Использовано: {ld.source_used}")
```

`TOP10_POPULAR` — кортеж из десяти наиболее популярных тикеров.

## 4. volatility.py

Модели волатильности.

### Базовые функции

| Функция | Назначение |
|---|---|
| `log_returns(prices)` | Логарифмические доходности |
| `simple_returns(prices)` | Простые доходности |
| `annualize_vol(daily_vol, periods=365)` | Аннуализация |

```python
from crypto_risk.volatility import log_returns, annualize_vol
r = log_returns(prices)
sigma_year = annualize_vol(r.std())
```

### EWMA

| Функция | Назначение |
|---|---|
| `ewma_volatility(returns, lam=0.94)` | Ряд скользящей σ |
| `ewma_forecast(returns, lam=0.94)` | Точечный прогноз σ |
| `ewma_covariance(returns, lam=0.94)` | EWMA-матрица ковариаций |

```python
from crypto_risk.volatility import ewma_volatility, ewma_forecast
sigma = ewma_volatility(r["BTC"])
sigma_fc = ewma_forecast(r["BTC"])
print(f"Текущий прогноз σ: {sigma_fc:.4f}")
```

### Класс `GARCHModel`

Конструктор:
`GARCHModel(vol="GARCH", dist="t", p=1, q=1, o=0)`.

Параметры:
* `vol` — `"GARCH"`, `"GJR"` или `"EGARCH"`
* `dist` — `"normal"`, `"t"` или `"skewt"`
* `p`, `q`, `o` — порядки модели

Метод `fit(returns, horizon=1)` возвращает структуру `GARCHResult` со
следующими полями:

| Поле | Назначение |
|---|---|
| `sigma` | Ряд условной σ во времени |
| `forecast_var` | Массив прогнозов дисперсии на горизонт |
| `params` | Параметры подгонки |
| `dist`, `vol`, `nu` | Метаданные модели |
| `aic`, `bic` | Информационные критерии |
| `model_obj` | Объект модели для дальнейших прогнозов |

Методы результата:
* `forecast_sigma_path()` — прогноз σ как массив
* `horizon_sigma(h)` — σ для совокупной доходности на горизонте h

```python
from crypto_risk.volatility import GARCHModel
g = GARCHModel(vol="GJR", dist="t").fit(r["BTC"], horizon=10)
print(f"Параметр ν: {g.nu:.2f}, BIC: {g.bic:.1f}")
print(f"σ на 10 дней: {g.horizon_sigma(10):.4f}")
```

### `select_best_garch(returns, candidates=..., criterion="bic", horizon=1)`

Перебирает спецификации и выбирает лучшую.

```python
from crypto_risk.volatility import select_best_garch
best, table = select_best_garch(r["BTC"], criterion="bic")
print(table.round(2))
print(f"Лучшая модель: {best.vol} + {best.dist}")
```

### Диапазонные оценки и HAR-RV

| Функция | Назначение |
|---|---|
| `parkinson_vol(ohlc)` | Простейшая диапазонная оценка |
| `garman_klass_vol(ohlc)` | С учётом разрыва open-close |
| `rogers_satchell_vol(ohlc)` | Корректна при ненулевом дрейфе |
| `yang_zhang_vol(ohlc, window=30)` | Минимально-дисперсионная оценка |
| `har_rv(realized_var)` | Каскадная модель прогноза дисперсии |

```python
from crypto_risk.volatility import yang_zhang_vol, har_rv, garman_klass_vol

yz = yang_zhang_vol(ohlc_df, window=30)
rv = (garman_klass_vol(ohlc_df) ** 2).dropna()
har = har_rv(rv)
print(f"R² модели HAR: {har.r2:.2f}")
print(f"Прогноз σ: {har.forecast_vol:.4f}")
```

## 5. var_es.py

Оценка мер риска. Соглашение: VaR и ожидаемые потери возвращаются как
положительные числа в долях капитала.

### Класс `RiskEstimate`

| Поле | Тип | Назначение |
|---|---|---|
| `var` | float | VaR |
| `es` | float | Ожидаемые потери |
| `method` | str | Название метода |
| `horizon` | int | Горизонт оценки в днях |
| `var_alpha` | float | Уровень доверия VaR |
| `es_alpha` | float | Уровень доверия ожидаемых потерь |
| `extra` | dict \| None | Дополнительные данные метода |

Метод `as_money(capital)` возвращает словарь с убытком в денежных единицах.

### Функции оценки

| Функция | Назначение |
|---|---|
| `historical_var_es(returns, var_alpha, es_alpha, horizon=1, overlap=False)` | Историческое моделирование |
| `parametric_var_es(mu, sigma, var_alpha, es_alpha, horizon=1, dist="normal", nu=None)` | Параметрический подход |
| `cornish_fisher_var_es(returns, var_alpha, es_alpha, horizon=1)` | Поправка Корниша-Фишера |
| `ewma_var_es(returns, var_alpha, es_alpha, horizon=1, lam=0.94, dist="normal", nu=None)` | Параметрический с EWMA-σ |
| `garch_var_es(garch_result, var_alpha, es_alpha, horizon=1, mu=0.0)` | На основе подогнанного GARCH |
| `fhs_var_es(returns, garch_result, var_alpha, es_alpha, horizon=1, n_paths=20000)` | Фильтрованное историческое |
| `monte_carlo_var_es(weights, mu, cov, var_alpha, es_alpha, horizon=1, n_sims=50000, dist="normal", nu=5)` | Метод Монте-Карло |

```python
from crypto_risk import var_es as ve

# Историческое моделирование
e1 = ve.historical_var_es(r["BTC"], 0.99, 0.975, horizon=1)
print(e1)

# Параметрический с распределением Стьюдента
e2 = ve.parametric_var_es(r["BTC"].mean(), r["BTC"].std(),
                          0.99, 0.975, 1, dist="t", nu=5)

# В денежных единицах
print(e1.as_money(capital=1_000_000))
```

### Декомпозиция риска

`component_var_es(weights, returns, var_alpha=0.99, es_alpha=0.975, method="gaussian")` —
возвращает `pandas.DataFrame` с колонками `weight`, `marginal_VaR`,
`component_VaR`, `pct_contrib_VaR`, `component_ES`, `pct_contrib_ES`.

```python
import numpy as np
weights = np.array([0.4, 0.3, 0.3])
attr = ve.component_var_es(weights, r[["BTC","ETH","SOL"]],
                            method="gaussian")
print(attr.round(4))
print(f"Главный источник риска: {attr['component_VaR'].idxmax()}")
```

`incremental_var(weights, returns, asset_idx, var_alpha=0.99)` — приращение
VaR от добавления одного актива.

### Бутстрэп на горизонт

`bootstrap_horizon_var_es(returns, var_alpha, es_alpha, horizon=10, block=1, n_paths=50000, seed=42)` —
оценка многодневного VaR через повторную выборку с возвращением.

```python
e_10d = ve.bootstrap_horizon_var_es(r["BTC"], 0.99, 0.975, horizon=10, block=1)
print(f"VaR за 10 дней: {e_10d.var:.4f}")
```

### Сводное сравнение

`compare_methods(returns, var_alpha, es_alpha, horizon=1, garch_result=None)` —
прогон всех методов в одной таблице.

```python
tbl = ve.compare_methods(r["BTC"], horizon=1, garch_result=g)
print(tbl.round(4))
```

## 6. evt.py

Теория экстремальных значений.

### `fit_gpd(losses, threshold=None, threshold_q=0.90, method="mle")`

Подгонка обобщённого распределения Парето. Возвращает структуру `GPDFit` с
полями `xi`, `beta`, `threshold`, `n_exceed`, `n_total`, `method` и свойством
`tail_index`.

```python
from crypto_risk import evt
import numpy as np

losses = -r["BTC"][r["BTC"] < 0].values
fit = evt.fit_gpd(losses, threshold_q=0.90)
print(f"ξ={fit.xi:.3f}, β={fit.beta:.3f}, индекс хвоста={fit.tail_index:.2f}")
```

### `pot_var_es(returns, var_alpha=0.99, es_alpha=0.975, threshold_q=0.90, method="mle")`

VaR и ожидаемые потери методом превышений над порогом. Возвращает
`RiskEstimate`.

```python
est = evt.pot_var_es(r["BTC"], 0.99, 0.975)
print(est)
```

### `conditional_evt(returns, garch_result, var_alpha=0.99, es_alpha=0.975, threshold_q=0.90)`

Условный вариант: GARCH плюс метод превышений на стандартизованных остатках.

```python
g = vol.GARCHModel("GJR", "t").fit(r["BTC"], horizon=1)
est = evt.conditional_evt(r["BTC"], g, 0.99, 0.975)
```

### Диагностики

| Функция | Назначение |
|---|---|
| `hill_estimator(losses, k=None)` | Оценка показателя тяжести хвоста |
| `mean_excess_plot_data(losses, n_points=40)` | Данные для графика средних превышений |

```python
hill = evt.hill_estimator(losses)
print(f"α по Хиллу: {hill:.2f}")

us, me = evt.mean_excess_plot_data(losses)
# us — пороги, me — средние превышения; пригодно для plot
```

## 7. covariance.py

Устойчивые оценки матрицы ковариаций.

### Оценки ковариации

| Функция | Назначение | Возвращает |
|---|---|---|
| `sample_cov(returns, annualize=False)` | Выборочная оценка | numpy.ndarray |
| `ledoit_wolf_cov(returns, annualize=False)` | Сжатие к скалярной | (cov, δ) |
| `oas_cov(returns, annualize=False)` | Oracle Approximating Shrinkage | (cov, δ) |
| `constant_correlation_shrinkage(returns, annualize=False)` | Сжатие к постоянной корреляции | (cov, δ) |
| `rmt_denoise_cov(returns, annualize=False)` | Очистка по Марченко-Пастуру | numpy.ndarray |

```python
from crypto_risk import covariance as cvm

cov_s = cvm.sample_cov(rets, annualize=True)
cov_lw, delta = cvm.ledoit_wolf_cov(rets, annualize=True)
print(f"Интенсивность сжатия δ: {delta:.3f}")

print(f"Обусловленность выборочной: {cvm.condition_number(cov_s):.0f}")
print(f"Обусловленность сжатой:     {cvm.condition_number(cov_lw):.0f}")
```

### Динамические условные корреляции

`dcc_garch(returns, a0=0.02, b0=0.95)` возвращает `DCCResult` с полями:

| Поле | Назначение |
|---|---|
| `cond_cov_last` | Условная ковариация на конец выборки |
| `cond_corr_last` | Условная корреляция на конец выборки |
| `a`, `b` | Параметры модели |
| `avg_corr_path` | Ряд средней попарной корреляции во времени |
| `names` | Имена активов |

Метод `annualized_cov()` возвращает аннуализированную условную ковариацию.

```python
dcc = cvm.dcc_garch(rets)
print(f"a={dcc.a:.3f}, b={dcc.b:.3f}, a+b={dcc.a+dcc.b:.3f}")
print(f"Средняя корреляция сейчас: {dcc.avg_corr_path.iloc[-1]:.2f}")
```

## 8. backtesting.py

Обратная проверка моделей риска. Каждый тест возвращает структуру
`BacktestResult(name, statistic, p_value, reject_h0, detail)`. Метод
`verdict(level=0.05)` выдаёт текстовое заключение.

### Получение пробоев

`get_violations(returns, var)` — массив индикаторов пробоев.

```python
from crypto_risk import backtesting as bt
I = bt.get_violations(realized, var_series)
print(f"Число пробоев: {int(I.sum())}")
```

### Тесты VaR

| Функция | Тестируемая гипотеза |
|---|---|
| `kupiec_pof(violations, var_alpha=0.99)` | Безусловное покрытие |
| `christoffersen(violations, var_alpha=0.99)` | Независимость и условное покрытие (возвращает словарь) |
| `duration_test(violations, var_alpha=0.99)` | Распределение длительностей по Вейбуллу |
| `dq_test(returns, var, var_alpha=0.99, lags=4)` | Динамический квантильный тест |
| `berkowitz_tail(returns, var, sigma=None, var_alpha=0.99, seed=42)` | Преобразование к равномерному распределению |

```python
print(bt.kupiec_pof(I, 0.99))
print(bt.christoffersen(I, 0.99)["conditional_coverage"])
print(bt.dq_test(realized, var_series, 0.99))
```

### Тесты ожидаемых потерь

| Функция | Тест |
|---|---|
| `acerbi_szekely_test1(returns, var, es, es_alpha=0.975, n_boot=5000, seed=42)` | По числу пробоев |
| `acerbi_szekely_es(returns, var, es, es_alpha=0.975, n_boot=5000, seed=42)` | Второй критерий |
| `acerbi_szekely_test3(returns, dist_cdf, es_alpha=0.975)` | Ранговый |

```python
print(bt.acerbi_szekely_test1(realized, var_series, es_series, 0.975))
```

### Сводный прогон и сигнальная схема

`run_var_backtests(returns, var, var_alpha=0.99)` возвращает таблицу всех
тестов.

`traffic_light(p_value)` возвращает строку `"GREEN"`, `"YELLOW"` или `"RED"`.

`model_risk_metrics(var_estimates: dict, n_violations=None, n_obs=None)` — оценка
модельного риска как разброса между методами + зона Базеля.

```python
table = bt.run_var_backtests(realized, var_series, 0.99)
print(table)

mr = bt.model_risk_metrics(
    {"hist": 0.05, "garch": 0.06, "evt": 0.07},
    n_violations=12, n_obs=1000)
print(f"Зона: {mr['basel_zone']}, множитель: {mr['basel_multiplier']}")
```

## 9. portfolio.py

Оптимизация портфеля.

### Базовые входы

`mean_cov(returns, annualize=True)` возвращает кортеж `(μ, Σ)`.

```python
from crypto_risk import portfolio as pf
mu, cov = pf.mean_cov(rets, annualize=True)
```

### Беты

`estimate_betas(returns, market="BTC")` возвращает `BetaEstimates` с полями
`betas`, `adjusted_betas` (по Блюму), `alphas`, `resid_var`, `market_var`.

`beta_covariance(beta_est, use_adjusted=False, annualize=True)` — ковариация
по однофакторной модели.

```python
be = pf.estimate_betas(rets, market="BTC")
print(be.betas.round(2))
cov_b = pf.beta_covariance(be, use_adjusted=True, annualize=True)
```

### Класс `PortfolioOptimizer`

Конструктор: `PortfolioOptimizer(mu, cov, rf=0.0, names=None)`. Методы:

| Метод | Описание |
|---|---|
| `min_variance(target_return=None, constraint="long_only", short_limit=0.25, min_w=0.02)` | Минимальная дисперсия |
| `max_sharpe(constraint="long_only", short_limit=0.25, min_w=0.02)` | Максимальный коэффициент Шарпа |
| `max_risk(constraint="long_short", short_limit=0.25, cap=None)` | Наиболее рискованный |
| `risk_parity()` | Равный вклад в риск |
| `efficient_frontier(n_points=50, constraint="long_only", short_limit=0.25, min_w=0.02)` | Список точек границы |
| `monte_carlo_frontier(n_portfolios=20000, constraint="long_only", seed=42)` | Облако случайных портфелей |

Допустимые значения `constraint`: `"long_short"`, `"short_limit"`,
`"long_only"`, `"min_weight"`.

Каждый метод возвращает `FrontierPoint(weights, ret, vol, sharpe)`.

```python
opt = pf.PortfolioOptimizer(mu, cov, names=list(UNIVERSE))

gmv = opt.min_variance(constraint="long_only")
msr = opt.max_sharpe(constraint="long_only")
rp  = opt.risk_parity()

front = opt.efficient_frontier(40, "short_limit", short_limit=0.25)
mc = opt.monte_carlo_frontier(15000, constraint="long_only")

print(f"Касательный портфель: σ={msr.vol:.2f}, доходность={msr.ret:.2f}")
for name, w in zip(UNIVERSE, msr.weights):
    if w > 0.01:
        print(f"  {name}: {w:.1%}")
```

### Two-fund theorem

`check_two_fund_theorem(mu, cov, tol=1e-6)` — численная проверка.

```python
tf = pf.check_two_fund_theorem(mu, cov)
print(f"Расхождение: {tf['max_weight_diff']:.2e}")
```

## 11. dependence.py

Копулы и хвостовая зависимость.

### Преобразования

| Функция | Назначение |
|---|---|
| `pseudo_observations(returns)` | Ранговое преобразование |
| `kendall_tau_matrix(returns)` | Матрица τ Кендалла |
| `empirical_tail_dependence(u, v, q=0.05)` | Пара (нижняя, верхняя) |
| `tail_dependence_matrix(returns, q=0.05, which="lower")` | Матрица |

```python
from crypto_risk import dependence as dep

```

### Копулы

Гауссова: `GaussianCopula.fit(returns)` возвращает объект с полями `corr`,
`names`. Метод `simulate(n, seed=42)` — выборка равномерных маргиналов.
Свойство `lower_tail_dependence` — матрица нулей.

Стьюдентовская: `StudentTCopula.fit(returns, nu_grid=...)` — поля `corr`,
`nu`, `names`. Методы `simulate(n, seed=42)`, `lower_tail_dependence_pair(rho)`,
свойство `lower_tail_dependence`.

```python
t = dep.StudentTCopula.fit(rets)
print(f"Параметр ν: {t.nu:.2f}")

# Хвостовая зависимость пары
print(f"λ для двух активов с ρ=0.7: {t.lower_tail_dependence_pair(0.7):.3f}")
```

### Симуляция портфеля через копулу

`copula_portfolio_returns(returns, weights, copula, n_sims=50000, seed=42)` —
возвращает массив доходностей портфеля. Зависимость берётся из копулы,
маргиналы — эмпирические.

```python
sim = dep.copula_portfolio_returns(rets, weights, t, n_sims=40000)
import numpy as np
var_99 = -np.quantile(sim, 0.01)
print(f"VaR 99% через t-копулу: {var_99:.4f}")
```

### Сравнение моделей зависимости

`compare_dependence_models(returns, q=0.05)` — таблица «эмпирическая,

## 14. engine.py — оркестратор `RiskEngine`

Главный объект пакета. Конструктор: `RiskEngine(config=None, data_mode="auto")`.
После создания — атрибуты `prices`, `returns`, `data_source`, `vol_forecaster`.

### Загрузка данных

| Метод | Назначение |
|---|---|
| `load_data(start, end, timeframe="1d")` | Через простой загрузчик |

```python
from crypto_risk import RiskEngine, RiskConfig

eng = RiskEngine(RiskConfig(universe=("BTC","ETH","SOL")))
```

## Дополнительно: вызов из существующих моделей

Если у пользователя уже есть свои оценки VaR или своя ковариационная матрица,
их можно подать в существующие функции напрямую. Например, чтобы прогнать
обратную проверку собственного ряда оценок:

```python
from crypto_risk import backtesting as bt
table = bt.run_var_backtests(my_realized, my_var_series, var_alpha=0.99)
```

Или построить эффективную границу по собственным μ и Σ:

```python
from crypto_risk.portfolio import PortfolioOptimizer
opt = PortfolioOptimizer(my_mu, my_cov, names=my_tickers)
front = opt.efficient_frontier(40, "long_only")
```

Это обеспечивает интеграцию с любыми внешними моделями без переписывания
пакета.
