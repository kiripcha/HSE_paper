# risk_managment_prod — production-сборка модуля управления рисками

Готовый к развёртыванию модуль рыночного риск-менеджмента для крипто-портфеля.
В составе: пакет `crypto_risk/`, ноутбуки с экспериментами, дымовые тесты,
демонстрационные сценарии, документация и Docker-обёртка.

## Структура

```
risk_managment_prod/
├── crypto_risk/             # производственный пакет управления рисками
├── notebooks/               # ноутбуки с экспериментами
│   ├── Crypto_Portfolio_Risk_Management.ipynb   # основная исследовательская тетрадь
│   ├── Prepare_data_task_1_3.ipynb              # подготовка данных, прежний пайплайн
│   ├── PRM_Ex_4-8.ipynb                         # эксперименты по риску
│   └── sec_part_RM_project_to_4.ipynb           # вторая часть проекта
├── tests/                   # дымовые тесты (16 функций, ~30 с прогон)
├── examples/                # демонстрационный сценарий
├── documentation/           # архитектурный обзор, методология, хэндофф LLM
├── data_cache/              # кэшированные котировки Binance с 2015 по 2025
├── Dockerfile               # сборка образа
├── docker-compose.yml       # оркестрация сервисов
├── .dockerignore
├── requirements.txt         # зависимости (диапазоны версий)
├── requirements-lock.txt    # зафиксированные точные версии (для воспроизводимости)
├── build_notebook.py        # генератор основного ноутбука
└── README.md
```

## Способ 1. Запуск через Docker (рекомендуется)

### Сборка образа

```bash
docker build -t crypto-risk:latest .
```

Сборка занимает 3–5 минут на первом запуске (установка научного стека).
Размер итогового образа около 2,5 ГБ.

### Запуск Jupyter Lab

```bash
docker run --rm -p 8888:8888 \
    -v $(pwd)/data_cache:/app/data_cache \
    -v $(pwd)/notebooks:/app/notebooks \
    crypto-risk:latest
```

После запуска откройте в браузере `http://localhost:8888`. Ноутбуки доступны
в папке `notebooks/`.

### Запуск дымовых тестов

```bash
docker run --rm crypto-risk:latest python tests/test_smoke.py
```

Ожидаемый результат — шестнадцать строк вида `OK ...` и итоговое
`ВСЕ ДЫМОВЫЕ ТЕСТЫ ПРОЙДЕНЫ`.

### Запуск демонстрационного сценария

```bash
# С реальными данными (требует сети при первом запуске)
docker run --rm -v $(pwd)/data_cache:/app/data_cache crypto-risk:latest \
    python examples/demo.py

# Полностью оффлайн (синтетические данные)
docker run --rm crypto-risk:latest python examples/demo.py --synthetic
```

## Способ 2. Запуск через docker-compose

```bash

docker compose up jupyter

docker compose run --rm tests

# Запустить демонстрацию
docker compose run --rm demo

docker compose run --rm rebuild-notebook
```

## Способ 3. Запуск без Docker (локальное окружение)

```bash
# Создание виртуального окружения
python3 -m venv .venv
.venv/bin/python -m pip install --upgrade pip

# Установка зависимостей — фиксированных версий для воспроизводимости
.venv/bin/python -m pip install -r requirements-lock.txt

# Альтернатива: диапазоны версий (последние совместимые)
# .venv/bin/python -m pip install -r requirements.txt

# Дымовые тесты
.venv/bin/python tests/test_smoke.py

# Демонстрация
.venv/bin/python examples/demo.py

.venv/bin/jupyter lab --notebook-dir notebooks
```

## Быстрая проверка работоспособности после установки

```bash
# 1. Импорт пакета
python -c "import crypto_risk; print(crypto_risk.__version__)"

# 2. Дымовые тесты (около 30 секунд, должны все пройти)
python tests/test_smoke.py

# 3. End-to-end демонстрация в синтетическом режиме (не требует сети)
python examples/demo.py --synthetic
```

Если все три шага проходят без ошибок — модуль установлен корректно.

## Интеграция в торговую систему

Минимальный пример встраивания контура управления рисками в действующую
торговую систему приведён в `documentation/AGENT_CONTEXT.md`, разделы 4–5.
В частности, разделы 5.1–5.6 содержат пошаговые рецепты:

* подключение модели глубокого обучения к подсистеме прогноза волатильности;
* предторговый контроль через `pre_trade_check`;
* добавление нового источника данных;
* бэктестинг собственной модели VaR через готовую батарею тестов.

## Ограничения и замечания

* Пакет рассчитан на рыночный риск; кредитный, операционный и регуляторный
  риски не охвачены и должны учитываться отдельными подсистемами.
* Для работы с реальными данными при первом запуске требуется сеть.
  При отсутствии сети система автоматически переходит на детерминированный
  синтетический генератор.
* Для крупных портфелей (более 50 активов) подгонка модели динамических
  условных корреляций может занимать несколько минут. В горячем контуре
  торговой системы рекомендуется использовать выборочную или сжатую оценку
  ковариационной матрицы.

## Контакты

Дальнейшая разработка и интеграция модуля — в соответствии с порядком,
описанным в файле `documentation/AGENT_CONTEXT.md`. Все изменения пакета
должны сопровождаться обновлением дымовых тестов и проверкой на их
прохождение.
