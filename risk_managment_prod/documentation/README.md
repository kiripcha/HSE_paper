# Документация `crypto_risk`

В этой папке три файла, каждый со своей аудиторией:

| Файл | Аудитория | Что внутри |
|---|---|---|
| [ARCHITECTURE.md](ARCHITECTURE.md) | разработчик, который меняет/расширяет код | модуль-за-модулем: назначение, публичный API, внутренняя логика, минимальный пример |
| [METHODOLOGY.md](METHODOLOGY.md) | риск-аналитик / ревьюер | для каждого метода: формула → код → обоснование выбора + ссылки на источники из `theory/` |
| [AGENT_CONTEXT.md](AGENT_CONTEXT.md) | другой LLM-агент при хэндоффе | самодостаточный брифинг: конвенции, полный API с сигнатурами, рецепты интеграции, сжатый теоретический ликбез, подводные камни |

С чего начать:
- **«хочу использовать»** → AGENT_CONTEXT, разделы 4-5.
- **«хочу понять, как устроено»** → ARCHITECTURE сверху вниз.
- **«хочу понять, почему так»** → METHODOLOGY по интересующей теме.
- **«хочу проверить, что ничего не сломал после правки»** → AGENT_CONTEXT, раздел 10 (smoke-сценарии).
